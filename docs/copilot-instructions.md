# Copilot Instructions

## Overview

This document provides high-level instructions and guidelines for code generation in this project. Follow these principles when implementing features or making changes to the codebase.

## Documentation Structure

The project documentation is organized as follows:

- **copilot-instructions.md** (this file) - High-level coding guidelines and principles
- **functional-requirements.md** - Core functional requirements for the application

## Coding Guidelines

### General Principles

- Write clean, maintainable, and well-documented code
- Follow established patterns and conventions within the project
- Prioritize readability and simplicity over complexity
- Ensure proper error handling and input validation

### Documentation Standards

- Keep documentation up-to-date with code changes
- Refer to [functional-requirements.md](./functional-requirements.md) for core application requirements
- Document any assumptions or design decisions

### Best Practices

- Write modular, reusable code
- Include appropriate comments for complex logic
- Follow the functional requirements specified in the documentation
